# Промпт для генерации диаграмм ВКR Mavix (улучшенная версия)

Адаптированный и проверенный на практике промпт для генерации UML/C4/ER/IDEF0/DFD
диаграмм в стиле ВКР. Учтены ошибки, всплывшие при работе (см. ❗). Базовые
соглашения и тулинг — в [RULES.md](RULES.md).

---

## ЗАДАЧА
Сгенерируй **самодостаточный SVG** для диаграммы:
[ТИП: UML Use Case / UML Class / UML Sequence / UML Deployment / C4 Context|Container|Component / ER (IDEF1X) / IDEF0 / DFD (Гейн–Сарсон)]

## ЦЕЛЬ И СОДЕРЖАНИЕ
[Перечисли списком всё: блоки/классы/акторы/процессы, связи, подписи на стрелках,
кратности. Лучше избыточно, чем неполно.]

## ФАЙЛ И РЕНДЕР
- Путь SVG: `assets/diagrams/<имя>.svg`; PNG: `assets/diagrams/png/<имя>.png`.
- ❗ **PNG рендерится через QtSvg** (`svg2png.py`, 2×). QtSvg **НЕ рендерит SVG
  `<marker>`** — наконечники рисуй **явными полигонами/линиями** (как
  `diagramlib.arrowhead`), иначе в PNG стрелка сольётся с границей/исчезнет.
- Шрифт: `font-family="Liberation Serif, Times New Roman, serif"` (Times-метрики).
- Подпись рисунка снизу по центру, italic 17 px: «Рисунок X – …», зазор ≥40 px
  от контента.

## СТИЛЬ (обязательно)
- Только чёрное на белом: `stroke:#000; fill:none;`. Без цветов/заливок/градиентов.
- Толщины: системная рамка 1.6; блоки/таблицы 1.1–1.2; разделители 0.7; стрелки 0.9–1.0.
- Шрифты: имя системы 18 bold; имя блока/таблицы 17 bold; шапка столбцов 12 bold;
  тип/описание 14–15 italic; подпись стрелки 14–15; актор 17.
- `viewBox` вмещает ВСЁ без обрезки; width/height = размерам viewBox.

## СТРЕЛКИ И СВЯЗИ (главные правила — здесь были ошибки)
- ❗ Наконечник стоит **СНАРУЖИ целевого блока с зазором ~12–14 px**, только
  указывает на блок — НИКОГДА не внутри и не поверх границы/текста.
- ❗ Связь не вести **вдоль грани** блока с «втыканием» в середину — заводить
  **перпендикулярно** (в верх/бок), чтобы наконечник явно указывал на блок.
- Маршрутизация ортогональная, 1 поворот; линия не пересекает текст.
- ❗ Подпись стрелки — РЯДОМ со стрелкой (отступ 8–12 px), НЕ на самой линии;
  длинную — в 2 строки.

## НОТАЦИЯ ПО ТИПАМ
- **ER (IDEF1X):** таблица = имя (italic) + шапка столбцов (Атрибут/Тип/Ограничения)
  + строки. Crow's foot: «1» — поперечная чёрточка, «N» — три лапки; ❗ метки
  **снаружи** таблицы (вдоль линии за краем). Колонку ограничений держать ≥100 px.
- **UML Class:** 3 секции (имя/атрибуты/операции). Композиция — **закрашенный ромб
  у «целого»**, БЕЗ наконечника на части. Агрегация — **полый ромб**. Ассоциация —
  простая линия + кратности; навигация — открытая стрелка. Зависимость — пунктир +
  открытая стрелка. Обобщение/реализация — **полый треугольник** (реализация —
  пунктир). Видимость: `+ - #`. Кратности на концах.
- **UML Use Case:** акторы — stick figure; сценарии — эллипсы; граница системы —
  прямоугольник. ❗ Связи актор—сценарий **ортогональные** («ствол» у актора +
  горизонтальные ветви), НЕ диагонали. Добавлять **«include»/«extend»** между
  сценариями (пунктир + открытая стрелка + подпись) — только в Use Case.
- **UML Sequence:** lifelines — вертикальный пунктир от прямоугольника участника.
  Синхронный вызов — сплошная стрелка с закрашенным наконечником; ответ — пунктир
  + открытый. Self-сообщение — петля; ❗ у крайнего участника петля и подпись —
  внутрь канвы (влево у правого), чтобы не обрезалось. Шаги нумеровать.
- **UML Deployment:** узлы — **3D-коробки** со стереотипом «device»/
  «executionEnvironment»; внутри — артефакты «artifact» (или вложенные «device»);
  между узлами — communication paths (сплошные) со стереотипом протокола («HTTPS»,
  «WebRTC», «UART»…) рядом с линией.
- **C4:** прямоугольники; имя (bold) + стек/технология (italic) `[Python, FastAPI]`;
  для внешних — пометка «(вне скоупа)». Уровни: Context → Container → Component.
- **IDEF0 (SADT):** функц. блоки лесенкой, номер A1..An в правом нижнем углу.
  ICOM: **Input — слева**, **Control — сверху**, **Output — справа**,
  **Mechanism — снизу** (стрелка вверх в низ блока). Подписи дуг — рядом, не на блоках.
- **DFD (Гейн–Сарсон):** процессы — скруглённые прямоугольники с номером; внешние
  сущности — прямоугольники; хранилища — **открытые справа** прямоугольники с
  ячейкой Dn; потоки — помеченные направленные стрелки. (Пересечения потоков
  допустимы, но текст не перекрывать.)

## РАСКЛАДКА БЕЗ НАЛОЖЕНИЙ ТЕКСТА
- ❗ Для табличных диаграмм (ER, Class) **генерируй раскладку программно**
  (ширина колонок по контенту: `font*0.6*len`, кириллица ×1.15) — тогда текст
  гарантированно не перекроется.
- НИ ОДИН текст не налезает на другой текст/рамку соседнего блока. НИ ОДНА линия
  не проходит сквозь текст.

## САМОПРОВЕРКА ПЕРЕД ВЫДАЧЕЙ (обязательно открыть PNG и посмотреть)
1. Наконечники СНАРУЖИ блоков, с зазором, видимы (полигоны, не `<marker>`)?
2. Нет линий сквозь текст? Нет налезающих подписей? Подписи рядом со стрелками, не на них?
3. Связи не идут вдоль грани и не втыкаются в середину рамки?
4. Crow's foot/метки «1» — снаружи таблиц? UML-нотация связей корректна (ромб/
   треугольник/кратности)?
5. Всё влезает в viewBox? Подпись рисунка с зазором ≥40 px?
6. Sequence — шаги по порядку, self-петли не обрезаны?

## ВЫХОД
Сохрани SVG, отрендери PNG, **открой PNG и визуально перепроверь пункты 1–6**.
Кратко (до 80 слов) опиши, что переставлял из-за пересечений.
